# 請假系統

一個專為小型單位設計的請假管理系統，使用 Google Sheets 作為資料庫，提供完整的請假申請、審核和管理功能。

## 系統特色

- 🏢 **適合小型單位** - 專為資源有限的小型組織設計
- 💰 **低成本營運** - 使用免費的 Google Sheets 和 GitHub Pages
- 👥 **雙角色設計** - 員工和管理員不同的操作介面
- 📱 **響應式設計** - 支援桌面和行動裝置
- 🔒 **權限管理** - 員工只能查看自己的資料
- 📊 **即時統計** - 管理員可查看系統統計和活動記錄

## 功能模組

### 員工功能
- 查看個人假期餘額
- 提交請假申請
- 查看請假紀錄
- 個人資料管理

### 管理員功能
- 員工資料管理
- 請假申請審核
- 系統統計查看
- 新增員工帳戶

## 技術架構

### 前端技術
- **HTML5** - 頁面結構
- **CSS3** - 樣式和響應式設計
- **JavaScript (Vanilla)** - 互動功能和資料處理

### 後端技術 (規劃中)
- **Google Apps Script** - 後端 API 和資料處理
- **Google Sheets** - 資料庫

### 部署平台
- **GitHub Pages** - 免費靜態網站託管
- **Netlify** - 替代託管選項

## 資料庫設計

### Google Sheets 資料表

#### 1. 員工基本資料表
| 欄位 | 類型 | 說明 |
|------|------|------|
| 員工ID | 文字 | 唯一識別碼 |
| 姓名 | 文字 | 員工姓名 |
| Email | 文字 | 登入帳號 |
| 到職日 | 日期 | 計算特休依據 |
| 特休天數 | 數字 | 剩餘特休 |
| 補休天數 | 數字 | 剩餘補休 |
| 部門 | 文字 | 所屬部門 |
| 狀態 | 文字 | 在職/離職 |

#### 2. 請假紀錄表
| 欄位 | 類型 | 說明 |
|------|------|------|
| 申請ID | 文字 | 唯一識別碼 |
| 員工ID | 文字 | 關聯員工 |
| 請假日期 | 日期 | 請假日期 |
| 時段 | 文字 | 全天/上午/下午 |
| 假別 | 文字 | 特休/補休/事假等 |
| 天數 | 數字 | 請假天數 |
| 原因 | 文字 | 請假原因 |
| 狀態 | 文字 | 待審核/已核准/已拒絕 |
| 申請時間 | 日期時間 | 提交時間 |
| 審核時間 | 日期時間 | 審核時間 |
| 審核者 | 文字 | 審核人員 |

#### 3. 假期類型表
| 欄位 | 類型 | 說明 |
|------|------|------|
| 假別代碼 | 文字 | 唯一識別碼 |
| 假別名稱 | 文字 | 顯示名稱 |
| 是否扣薪 | 布林 | 是否扣薪 |
| 最大天數 | 數字 | 年度上限 |

## 使用流程

### 員工請假流程
1. 員工登入系統
2. 查看個人假期餘額
3. 填寫請假申請表單
4. 提交申請等待審核
5. 查看申請狀態和結果

### 管理員審核流程
1. 管理員登入系統
2. 查看待審核申請列表
3. 審核請假申請
4. 核准或拒絕申請
5. 系統自動更新假期餘額

## 安裝和部署

### 1. 本地開發
```bash
# 克隆專案
git clone [repository-url]
cd dayoff

# 使用本地伺服器運行
python -m http.server 8000
# 或使用 Node.js
npx serve .
```

### 2. 部署到 GitHub Pages
1. 將專案推送到 GitHub 倉庫
2. 在倉庫設定中啟用 GitHub Pages
3. 選擇分支和資料夾
4. 訪問生成的網址

### 3. 部署到 Netlify
1. 註冊 Netlify 帳戶
2. 連接 GitHub 倉庫
3. 設定建置選項
4. 部署完成

## 設定 Google Sheets

### 1. 建立 Google Sheets
1. 建立新的 Google Sheets 文件
2. 建立三個工作表：員工資料、請假紀錄、假期類型
3. 設定欄位標題

### 2. 設定 Google Apps Script
1. 在 Google Sheets 中開啟 Apps Script
2. 建立 API 端點
3. 設定 CORS 和權限

### 3. 更新前端設定
1. 更新 API 端點 URL
2. 測試資料連線
3. 部署更新

## 安全性考量

- 使用 HTTPS 加密傳輸
- 實作適當的認證機制
- 限制 API 存取權限
- 定期備份資料
